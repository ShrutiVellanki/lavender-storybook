# Contributing

How to add or modify components in the Lavender Design System.

---

## Adding a New Component

### 1. Create the component directory

```
src/components/ui/MyComponent/
├── MyComponent.tsx        # Component implementation
├── MyComponent.types.ts   # TypeScript interfaces
├── MyComponent.styles.ts  # Variant styles (if using cva pattern)
└── index.ts               # Public exports
```

### 2. Follow existing conventions

- Use `cn()` from `@/lib/utils` for class merging
- Source colors from CSS custom properties (`var(--primary)`, etc.) — never hardcode hex values
- Forward refs with `React.forwardRef` on all DOM-wrapping components
- Export types alongside the component from `index.ts`
- Props live in `ComponentName.types.ts`, never inline in the `.tsx` file

### 3. Write stories

Create `stories/my-component.stories.tsx` with at minimum:

| Story | Purpose |
|-------|---------|
| **Playground** | All props wired to Storybook controls via `args` |
| **Feature stories** | One story per visual variant, state, or behavior |
| **Recipe** (optional) | Combine with other components in a real-world layout |

```tsx
import type { Meta, StoryObj } from "@storybook/react"
import { MyComponent } from "@/components/ui/MyComponent"

const meta: Meta<typeof MyComponent> = {
  title: "Category/MyComponent",
  component: MyComponent,
  tags: ["autodocs"],
  parameters: { layout: "centered" },
}
export default meta
type Story = StoryObj<typeof MyComponent>

export const Playground: Story = {
  args: { /* default prop values */ },
}
```

### 4. Choose the right sidebar group

| Group | When to use |
|-------|------------|
| `Inputs/` | Buttons, selects, text fields, toggles — anything that captures user input |
| `Layout/` | Containers, modals, accordions, tabs — structural components |
| `Data Display/` | Tables, lists, charts, stat cards — components that render data |
| `Information/` | Badges, progress bars, loaders, skeletons, error states |
| `Forms/` | Multi-field composite forms |
| `Theme/` | Theme providers, switchers, and utilities |

### 5. Add interaction tests

Every Playground story should include a `play` function:

```tsx
import { expect, userEvent, within } from "@storybook/test"

export const Playground: Story = {
  args: { children: "Click me" },
  play: async ({ canvasElement }) => {
    const canvas = within(canvasElement)
    const button = canvas.getByRole("button", { name: "Click me" })
    await userEvent.click(button)
    await expect(button).toBeInTheDocument()
  },
}
```

**Querying elements** — use queries scoped via `within(canvasElement)`:

| Query | When to use |
|-------|-------------|
| `getByRole("button", { name: "..." })` | Interactive elements — **preferred** |
| `getByText(/pattern/i)` | Visible text content |
| `getByPlaceholderText("...")` | Inputs without `htmlFor` labels |
| `getByLabelText("...")` | Inputs with associated labels |
| `findByText("...")` | Text that appears asynchronously |

**Simulating user actions:**

```tsx
await userEvent.click(element)
await userEvent.type(input, "hello")
await userEvent.keyboard("{Enter}")
```

**Asserting state:**

```tsx
await expect(element).toBeVisible()
await expect(element).toHaveTextContent("Expected text")
await expect(element).toHaveAttribute("disabled")
```

**Waiting for async changes:**

```tsx
import { waitFor } from "@storybook/test"
await waitFor(() => expect(canvas.getByText("Updated")).toBeVisible())
```

**Testing callbacks with `fn()`:**

```tsx
import { fn } from "@storybook/test"

export const Playground: Story = {
  args: { onClick: fn() },
  play: async ({ canvasElement, args }) => {
    const canvas = within(canvasElement)
    await userEvent.click(canvas.getByRole("button"))
    await expect(args.onClick).toHaveBeenCalledOnce()
  },
}
```

## Modifying an Existing Component

1. Edit the component in `src/components/ui/ComponentName/`
2. Update types in `ComponentName.types.ts` if the API changed
3. Add or update stories to cover any new props or states
4. Check both **Lavender Dawn** and **Lavender Moon** themes in Storybook
5. Run `npm run test-storybook` to verify all interaction tests pass

---

## Code Style

- **TypeScript** — strict mode, no `any`, explicit prop types in `*.types.ts` files
- **Tailwind CSS** — use utility classes via `cn()`, never inline `style` objects for themeable properties
- **Naming** — PascalCase for components and directories, camelCase for utilities
- **Accessibility** — ARIA attributes on interactive components, keyboard navigation, focus management
