# Getting Started

This guide walks you through adding Lavender components to an existing React + Tailwind CSS project.

---

## Prerequisites

Your project needs:

- **React 18+**
- **Tailwind CSS 3.4+**
- **TypeScript** (recommended but not required)

---

## 1. Install Peer Dependencies

Lavender components use a small set of shared utilities:

```bash
npm install lucide-react clsx tailwind-merge tailwindcss-animate
```

| Package | Purpose |
|---------|---------|
| `lucide-react` | Icons used in Accordion chevrons, Modal close, Loading spinner, ErrorDisplay, StarRating, ThemeSwitcher, Select |
| `clsx` + `tailwind-merge` | The `cn()` utility for merging Tailwind classes |
| `tailwindcss-animate` | Animation utilities used by Loading spinner |

If you use the **Chart** components, also install:

```bash
npm install recharts
```

---

## 2. Add the `cn()` Utility

Create a utility file at `lib/utils.ts`:

```typescript
import { type ClassValue, clsx } from "clsx"
import { twMerge } from "tailwind-merge"

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

Ensure your project's path aliases map `@/` to the project root (or adjust imports).

## 4. Configure Tailwind

Make sure your `tailwind.config.ts` maps the CSS variables to Tailwind utilities:

```typescript
import type { Config } from "tailwindcss"

const config: Config = {
  darkMode: ["class"],
  content: ["./src/**/*.{js,ts,jsx,tsx}"],
  theme: {
    extend: {
      colors: {
        background: "var(--background)",
        foreground: "var(--foreground)",
        card: { DEFAULT: "var(--card)", foreground: "var(--card-foreground)" },
        popover: { DEFAULT: "var(--popover)", foreground: "var(--popover-foreground)" },
        primary: { DEFAULT: "var(--primary)", foreground: "var(--primary-foreground)" },
        secondary: { DEFAULT: "var(--secondary)", foreground: "var(--secondary-foreground)" },
        muted: { DEFAULT: "var(--muted)", foreground: "var(--muted-foreground)" },
        accent: { DEFAULT: "var(--accent)", foreground: "var(--accent-foreground)" },
        destructive: { DEFAULT: "var(--destructive)", foreground: "var(--destructive-foreground)" },
        border: "var(--border)",
        input: "var(--input)",
        ring: "var(--ring)",
      },
      borderRadius: {
        lg: "var(--radius)",
        md: "calc(var(--radius) - 2px)",
        sm: "calc(var(--radius) - 4px)",
      },
    },
  },
  plugins: [require("tailwindcss-animate")],
}

export default config
```

## 6. File Structure Inside a Component

Each component follows a consistent internal structure:

```
Button/
  Button.tsx          # Component implementation
  Button.types.ts     # Exported TypeScript interfaces
  Button.styles.ts    # Variant maps, style constants
  index.ts            # Barrel re-export
```

| File | Purpose | When to include |
|------|---------|-----------------|
| `ComponentName.tsx` | The React component itself | Always |
| `ComponentName.types.ts` | Exported interfaces, type aliases, enums | Always (even if small) |
| `ComponentName.styles.ts` | Variant class maps, reusable style constants | Only if the component has variant maps or shared style objects |
| `index.ts` | Re-exports all public symbols | Always |

**Example `index.ts`:**

```typescript
export { Button } from "./Button"
export type { ButtonProps, ButtonVariant, ButtonSize } from "./Button.types"
```

**Importing cleanly:**

```typescript
import { Button } from "@/components/ui/Button"
```

The barrel `index.ts` means you never need to reach into individual files.

## 7. Usage Example

```tsx
import { Button } from "@/components/ui/Button"
import { Card, CardHeader, CardTitle, CardContent } from "@/components/ui/Card"
import { Badge } from "@/components/ui/Badge"

export default function App() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>
          Dashboard <Badge variant="secondary">Beta</Badge>
        </CardTitle>
      </CardHeader>
      <CardContent>
        <p>Welcome to your dashboard.</p>
        <Button className="mt-4">Get Started</Button>
      </CardContent>
    </Card>
  )
}
```

## Component Dependencies

Some components depend on others or on the theme system:

| Component | Depends On |
|-----------|-----------|
| ThemeSwitcher | ThemeProvider |
| Chart (all) | recharts, ChartContext |
| ErrorDisplay | Button (optional, for retry) |

All other components are fully standalone.

The only shared dependencies across all components are:

- `@/lib/utils` — the `cn()` class merging utility
- `lucide-react` — icons
- `@/components/ui/ThemeProvider` — only needed by ThemeSwitcher
