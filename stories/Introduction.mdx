# Lavender

The design system and component library powering [**Lavender Finance**](https://github.com/ShrutiVellanki/lavender-finance) -- a personal net-worth tracking dashboard built with React, Vite, and Tailwind CSS.

Lavender provides **24 production-ready components**, two hand-crafted color themes (Lavender Dawn and Lavender Moon), and a complete set of design tokens. Every component is copy-pasteable, fully accessible, and theme-aware out of the box.

| | |
|---|---|
| **Framework** | React 18 |
| **Styling** | Tailwind CSS 3.4 |
| **Language** | TypeScript |
| **Docs** | Storybook 8 |
| **Icons** | Lucide React |
| **Charts** | Recharts |

---

## Used In: Lavender Finance

Lavender Finance is a personal finance dashboard that uses this component library for its entire UI. The app features net-worth tracking, account management, transaction history, budgeting, and interactive financial charts -- all rendered with components from this Storybook.

Toggle between **Lavender Dawn** (light) and **Lavender Moon** (dark) using the theme switcher in the Storybook toolbar above to preview every component in both themes.

## Components

### Core UI

| Component | Description |
|-----------|-------------|
| **Button** | Six variants (default, secondary, destructive, outline, ghost, link), four sizes |

## Architecture

All components follow these principles:

- **Zero external UI dependencies** -- built on native HTML, React, and Tailwind CSS. No Radix, Headless UI, or other UI primitives required at runtime.
- **Controlled and uncontrolled** -- interactive components accept both `value`/`onChange` (controlled) and `defaultValue` (uncontrolled) patterns.
- **Generic typing** -- Select, Autocomplete, Combobox, and VirtualizedList are generic (`<T>`) so they work with any data shape.
- **ARIA compliant** -- roles, states, and keyboard interaction patterns follow WAI-ARIA Authoring Practices.
- **Theme token based** -- all colors reference CSS custom properties, making theme swaps a single class toggle.
