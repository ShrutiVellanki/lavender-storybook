## Token Layers

The system uses three token layers that allow components to work across themes without any conditional logic:

| Layer | Example | Purpose |
|-------|---------|---------|
| **Semantic** | `--primary`, `--border`, `--muted` | Theme-aware values that swap automatically between Dawn and Moon |
| **Chart** | `--chart-1` through `--chart-5` | Data visualization colors tuned for contrast in both themes |
| **Extended** | `lavenderDawn.iris`, `lavenderMoon.foam` | Fixed-hex values for when a specific color is needed regardless of theme |

‚Üí See **Tokens / Getting Started** for the full palette and setup guide.

## Design Decisions

### Accessibility First
- All interactive components support full keyboard navigation
- Focus rings use `--ring` token for theme consistency
- ARIA attributes on compound components (Accordion, Tabs, Modal, Tooltip)
- Color contrast ratios meet WCAG AA in both themes

### Composable Architecture
- Each component is a self-contained directory (`ComponentName.tsx`, `ComponentName.types.ts`, `index.ts`)
- Variant logic separated into `*.styles.ts` files using `cva`-style patterns
- `cn()` utility enables safe class overrides without specificity wars

### Theme System
- CSS custom properties on `:root` / `.dark` ‚Äî zero JavaScript runtime cost for theming
- `ThemeProvider` context for programmatic control with `localStorage` persistence
- Storybook's `withThemeByClassName` decorator for live theme switching in docs

## Quick Start

```bash
npm install clsx tailwind-merge lucide-react tailwindcss-animate
```

1. Copy the component directory into your project
2. Add `cn()` utility and theme tokens (see **Tokens / Getting Started**)
3. Wrap your app in `ThemeProvider`
4. Use components ‚Äî they respond to the active theme automatically
